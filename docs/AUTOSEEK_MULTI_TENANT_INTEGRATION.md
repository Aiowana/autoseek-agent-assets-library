## 1. 需求概述

### 1.1 目标

在现有的 AutoSeek 平台上增加**多租户 Git 仓库集成**功能，使每个用户/项目能够：

1. **连接自己的 Git 仓库**（GitHub/Gitee）来管理 Agent 资产
2. **同步仓库中的工具/提示词/技能**到平台
3. **在平台内直接创建和管理**资产，并同步回 Git
4. **每个项目独立的资产库**，互不干扰

#### 表单验证规则

```typescript
interface GitConfigForm {
  platform: "github" | "gitee";
  repo: string;        // 格式: "owner/repo" 或完整 URL
  branch: string;       // 默认: "main"
  token: string;        // Personal Access Token
  webhook_secret: string;
}

// 验证规则
const validationRules = {
  repo: {
    required: true,
    pattern: /^[a-zA-Z0-9_-]+\/[a-zA-Z0-9_.-]+$/,
    message: "请输入有效的仓库地址 (owner/repo 格式)"
  },
  branch: {
    required: true,
    maxLength: 100
  },
  token: {
    required: true,
    minLength: 20,
    pattern: /^ghp_[a-zA-Z0-9]{36,}$/
  },
  webhook_secret: {
    required: true,
    minLength: 16
  }
};
```

#### API 调用

```typescript
// 保存仓库配置
POST /api/v1/projects/:id/git/config
{
  platform: "github" | "gitee",
  repo: "alpha-team/agent-assets",
  branch: "main",
  token: "ghp_xxx...",
  webhook_secret: "secret_xxx"
}

// 验证仓库配置
POST /api/v1/projects/:id/git/validate
{
  platform: "github",
  repo: "alpha-team/agent-assets",
  token: "ghp_xxx..."
}

// 响应
{
  valid: true,
  repo_exists: true,
  branch_exists: true,
  assets_detected: {
    tools: 3,
    prompts: 2,
    skills: 1
  },
  error?: string
}

// 获取仓库配置
GET /api/v1/projects/:id/git/config

// 删除仓库配置
DELETE /api/v1/projects/:id/git/config
```

## 9. 注意事项

### 9.1 安全性

- **Token 加密存储**：Git Token 必须在后端加密存储，不能暴露给前端
- **权限控制**：只有项目所有者/管理员可以配置 Git
- **Webhook 验证**：后端必须验证 Webhook 签名
- **API 访问控制**：所有 API 需要验证用户权限

### 9.2 兼容性

- **向后兼容**：不配置 Git 的项目仍能使用预定义工具包
- **渐进增强**：Git 资产是额外功能，不破坏现有流程
- **独立运行**：即使 Git 服务不可用，核心功能仍能正常运行

### 9.3 用户体验

- **首次引导**：首次进入资产库显示配置向导
- **错误处理**：Token 过期、网络错误等友好提示
- **实时反馈**：同步状态实时更新
- **性能优化**：大仓库的资产列表分页加载

### 9.4 技术考虑

- **WebSocket 集成**：同步状态通过 WebSocket 推送给前端
- **缓存策略**：Git 数据缓存，减少 API 调用
- **并发控制**：同步时避免重复操作
- **重试机制**：网络错误自动重试
